## AI引擎缺陷诊断 (Brain B联合)

### 已确认故障模式
1. **静默失败毒瘤**: `verifyBatchOneByOne`的`continue on error`不产出任何日志/指标
2. **JSON零值陷阱**: extractJSON成功但Unmarshal到空结构体 → ai_confidence=0 → 被当作"未处理"而非"AI判定benign"
3. **Context超时**: DeepSeek Reasoner 30-60s推理 vs 120s HTTP timeout vs scan-level context
4. **Data race**: 3并发workers + shared slice 无mutex
5. **Batch prompt截断**: 15条finding的JSON可能超出token限制 → 返回不完整
6. **缺少启动自检**: 引擎启动前不验证"1条known finding能否成功处理"
7. **Triage阶段零可见性**: 不知道triage的42条输入有多少到达DeepVerify

### 架构根本问题
1. **Best-effort, no feedback loop**: AI引擎失败后系统报告"partial"继续运行
2. **零prompt/response日志**: 无法回溯API交互
3. **ai_confidence=0语义过载**: API失败/JSON解析失败/AI判定benign → 共用一个值
4. **无hard failure boundary**: batch全挂不应回退到逐条，应直接报错
