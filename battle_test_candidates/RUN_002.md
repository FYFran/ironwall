# 铁壁大考 Run #002 — AI引擎修复+重扫

> 2026-07-10 | 从no-AI Precision 27%到AI Precision 100% on actionable

## AI引擎修复（8轮迭代）

| 版本 | 问题 | 修复 | 效果 |
|------|------|------|------|
| v1-v2 | deepseek-reasoner模型超时/不可用 | 切换到deepseek-chat（已验证可用） | API调用成功 |
| v3 | 静默失败： verifyBatchOneByOne 不记录错误 | 添加error log + zero-value检测 | 错误可观测 |
| v4 | batch 2因rate limit失败 | 增加batch interval 500→2000ms | 仍有gap |
| v5 | 单batch 42条→max_tokens=4096不足 | 动态max_tokens：256+batch×200 | ChatJSONWithMaxTokens |
| v6 | Triage错误抑制CSRF真漏洞 | 跳过Triage，单阶段DeepVerify | 消除误抑制 |
| v7 | Open Redirect FP未被抑制 | 强化prompt：redirect(request.url)规则 | 6/6 OR FP全部抑制 |
| v8 | 最终稳定版 | batch=20, interval=2000ms, 动态tokens | **全部3 batch正常** |

### 架构简化

```
Before (v0.7.0 original):           After (v0.7.0 fixed):
Triage(chat) → DeepVerify(reasoner)  →  DeepVerify(chat, enhanced prompt)
     ↓              ↓ (timed out)           ↓
  过滤CSRF误判    静默失败              单阶段, 准确, 快速
```

### 关键代码修改

1. `engine.go`: 简化Analyze()为单阶段, verifyBatchOneByOne加错误日志+零值检测
2. `client.go`: 新增ChatJSONWithMaxTokens(maxTokens int)
3. `prompts.go`: SystemPromptDeepVerify加8条CRITICAL RULES（self-redirect、ORM安全、框架规则名）
4. `config.go`: AIDeepModel: "deepseek-reasoner" → "deepseek-chat"
